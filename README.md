## ğŸ“ Interview Tips

### Before the Interview
1. **Practice** coding these from scratch (not copy-paste)
2. **Understand** the core concepts, not just the syntax
3. **Time yourself** - can you complete in 30-60 min?
4. **Explain** your approach out loud

### During the Interview
1. **Start simple** - get basic version working first
2. **Ask clarifying questions** - understand requirements
3. **Think aloud** - explain your thought process
4. **Test as you go** - don't wait until the end
5. **Handle edge cases** - show you think about them

### Common Interview Questions
- "Why did you choose this approach?"
- "How would you optimize this for production?"
- "What if we had 10,000 items instead of 10?"
- "How would you test this?"
- "What would you change if you had more time?"
